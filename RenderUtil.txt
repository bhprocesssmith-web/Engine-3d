O termo exato que você procura é provavelmente
"RenderUtil" (com "U" maiúsculo) ou algo similar, que significa "Utilitários de Renderização".
Diferente de Matrix4f e Quaternion, que são tipos de dados matemáticos universais na computação gráfica, "RenderUtil" não é um termo 
técnico padrão ou uma classe universal em todas as engines. É um nome de classe ou módulo específico que um desenvolvedor ou a equipe 
de uma engine decidiu criar para agrupar funções auxiliares.
O que faz um "RenderUtil"?
Em essência, uma classe ou biblioteca chamada "RenderUtil" dentro de uma engine 3D é um conjunto de ferramentas e funções de conveniência 
que simplificam tarefas comuns e repetitivas relacionadas ao processo de renderização.
Essas funções geralmente lidam com a "cola" necessária para fazer a renderização funcionar, como:

    Carregamento de Shaders: Funções para carregar, compilar e vincular programas de shader GLSL/HLSL a partir de arquivos.
    Gerenciamento de Texturas: Utilitários para carregar imagens de disco, criar IDs de textura na GPU e configurar parâmetros de 
filtragem (como mipmaps).
    Configuração do Estado Gráfico: Funções simples para ativar ou desativar rapidamente recursos da GPU, como blending (transparência), 
depth testing (teste de profundidade para saber o que está na frente) ou face culling (não renderizar a parte de trás dos objetos).
    Desenho de Primitivas Simples: Funções para desenhar formas básicas de depuração (debug), como caixas delimitadoras (bounding boxes), 
linhas ou esferas, que os programadores usam para visualizar dados internos do motor que não são visíveis no jogo final.
    Conversões de Formato: Funções para converter dados de um formato de CPU para formatos otimizados para a GPU. 

Em resumo, "RenderUtil" é a "caixa de ferramentas" do programador gráfico, cheia de pequenas ajudas para evitar a reescrita do mesmo código 
repetidamente ao configurar uma cena para ser desenhada na tela.
